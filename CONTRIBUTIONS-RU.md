## 📊 Чем этот форк отличается от оригинала

Этот форк представляет собой **полную трансформацию** оригинального проекта [realskyrin/android-i18n-mcp](https://github.com/realskyrin/android-i18n-mcp). Оригинальный репозиторий предоставлял базовый MCP-сервер для перевода Android strings.xml файлов. Этот форк был улучшен ИИ до **production-ready решения** с комплексным тестированием, усиленной безопасностью и функциями корпоративного уровня.

### 2. Усиление безопасности

- **Предотвращение Path Traversal** — Валидация всех путей относительно корня проекта
- **Защита от XSS** — HTML-эскейпинг во всех сообщениях об ошибках
- **Валидация входных данных** — Zod схема для всех параметров MCP инструментов
- **Санитизация фильтров файлов** — Блокировка опасных паттернов (`../`, `..\\`, `\0`, и др.)
- **Кастомные классы ошибок** — Типизированные ошибки с поддержкой cause

### 3. Новые функции

- **7 MCP инструментов:**
  - `translate_all_modules` — Перевести все strings.xml файлы в проекте
  - `translate_module` — Перевести конкретный модуль
  - `check_changes` — Проверить незакоммиченные изменения
  - `check_missing_languages` — Найти отсутствующие языковые директории
  - `create_and_translate_missing_languages` — Создать и перевести отсутствующие директории
  - `get_job_status` — Проверить статус фоновой задачи
  - `configure_logging` — Включить/выключить файловое логирование

- **Система отслеживания задач** — Персистентное хранение задач с timeout 60 секунд
- **Реалтайм прогресс** — Плавное обновление прогресса во время перевода
- **Проверка подключимости** — Верификация доступности API перед переводом
- **Режим принудительного обновления** — Повторный перевод существующих переводов
- **Поддержка мульти-модулей** — Обработка нескольких Android модулей

## ⚠️ Отказ от ответственности: ИИ-генерация

Этот репозиторий содержит код, который **существенно сгенерирован и улучшен ИИ**. В частности:

- **ИИ модель:** Roo Code (LLM-ассистент)
- **Период генерации:** 30 апреля — 16 мая 2026
- **Объём:** Все изменения кода, исправления багов, тесты и документация
- **Проверка человеком:** **ОБЯЗАТЕЛЬНА** перед развёртыванием в production

### Почему ИИ-код нуждается в проверке

1. **Корректность:** ИИ может создавать логически некорректный код, который проходит тесты
2. **Безопасность:** ИИ может упустить краевые случаи или внести уязвимости
3. **Производительность:** Оптимизации ИИ могут быть неоптимальными для всех сценариев
4. **Поддерживаемость:** Паттерны кода ИИ могут быть трудны для понимания людьми
5. **Соответствие:** Ваша организация может иметь политики использования ИИ-кода

### Рекомендуемые шаги проверки

1. **Аудит безопасности** — Проверьте всю валидацию входных данных и файловые операции
2. **Код-ревью** — Попросите старших разработчиков проверить критические пути
3. **Интеграционное тестирование** — Протестируйте с вашей реальной структурой Android проекта
4. **Тестирование производительности** — Проверьте производительность перевода с вашим API
5. **Проверка соответствия** — Убедитесь в соответствии политикам вашей организации

## 🚀 Начало работы

### Требования

- Node.js 18+
- API ключ OpenAI / DeepSeek / Anthropic
- Android проект с файлами strings.xml

### Установка

```bash
# Клонировать этот форк
git clone https://github.com/DScoNOIZ/android-i18n-mcp.git
cd android-i18n-mcp

# Установить зависимости
npm install

# Собрать
npm run build

# Настроить
cp .env.example .env
# Отредактировать .env с вашим API ключом
```

### MCP конфигурация

```json
{
  "mcpServers": {
    "android-i18n": {
      "command": "node",
      "args": ["/path/to/android-i18n-mcp/build/index.js"],
      "env": {
        "ANDROID_PROJECT_ROOT": "/path/to/your/android/project",
        "TRANSLATION_PROVIDER": "openai",
        "TRANSLATION_API_KEY": "your_api_key_here",
        "TRANSLATION_LANGUAGES": "zh-CN,es,fr,de,ja,ko,ru"
      }
    }
  }
}
```
